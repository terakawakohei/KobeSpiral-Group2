# ユースケース：イベントの登録をする 

## 概要
管理者がイベントの追加をする

## アクター
- 管理者

## 事前条件
- 管理者がイベント登録画面を表示させていること

## 事後条件
- データベースにイベントが登録されていること

## トリガ―
- 管理者がイベント登録画面にアクセスする 

## 基本フロー
1. 管理者はイベントの日時・場所を選択する
2. 管理者はイベントの詳細をフォームに入力する
3. 管理者は登録ボタンを押す
4. システムはイベントをデータベースに追加する
